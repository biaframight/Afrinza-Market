BMW-Sync still running fine. Now here's what to do to push just BMW-Sync to its own GitHub repo:

Step 1 — Create a new empty GitHub repo
Go to github.com → New repository → name it bmw-sync → don't add a README or .gitignore → Create.

Step 2 — Push just the BMW-Sync folder using git subtree

Open the Replit Shell and run these two commands (replace the URL with your new repo):

git remote add bmw-sync https://github.com/YOUR_USERNAME/bmw-sync.git

git subtree push --prefix=artifacts/bmw-sync bmw-sync main

git subtree push is a built-in git feature that takes one subfolder and pushes it as the root of a separate repo — so your bmw-sync GitHub repo will look like a clean standalone Next.js project, with no Afrinza code, no lib/ packages, nothing from the monorepo.

Step 3 — Vercel from that new GitHub repo

In Vercel: New Project → Import from GitHub → select bmw-sync → Root Directory = / (leave blank / root) (because it's now the repo root) → the vercel.json handles the rest.

Add the two env vars:

NEXT_PUBLIC_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_ANON_KEY
Deploy. Done.

Going forward, whenever you update BMW-Sync here in Replit, just run git subtree push --prefix=artifacts/bmw-sync bmw-sync main again to sync the changes.
